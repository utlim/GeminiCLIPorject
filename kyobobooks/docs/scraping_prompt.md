# 교보문고 컴퓨터/IT 베스트셀러 데이터 수집 계획서

이 문서는 교보문고의 컴퓨터/IT 베스트셀러 목록을 수집하기 위한 스크래핑 설계 및 구현 계획을 정의합니다. 수집된 데이터는 기존에 수집된 YES24 베스트셀러 데이터와 연계하여 통합 도서 시장 분석 및 시각화 자료 구축에 사용됩니다.

---

## 1. 수집 개요

- **수집 대상**: 교보문고 국내도서 > 컴퓨터/IT 카테고리의 베스트셀러 목록
- **수집 빈도**: 단회성 전체 수집 (이후 필요 시 일간/주간 스케줄로 확장 가능)
- **최종 출력 파일**: `kyobobooks/data/bestsellers.csv`
- **파일 형식 및 인코딩**: CSV 포맷, `utf-8-sig` 인코딩 (Excel에서 한글 깨짐 방지)

---

## 2. 수집 데이터 필드 정의 (Schema)

YES24 분석 데이터와의 결합 및 비교 분석을 위해 동일하거나 호환되는 데이터 구조를 유지합니다.

| 필드명 (CSV Header) | 데이터 타입 | 설명 | 교보문고 대응 항목 및 비고 |
| :--- | :--- | :--- | :--- |
| `goods_no` | string | 도서 고유 번호 | 교보문고 상품 고유 ID (예: 바코드 번호 혹은 itemId) |
| `rank` | int | 베스트셀러 순위 | 1위부터의 순차적 순위 |
| `title` | string | 도서명 | 메인 도서 제목 (개행 문자 및 연속 공백 정화) |
| `subtitle` | string | 부제목 | 도서의 부제 (존재할 경우) |
| `author` | string | 저자 | 저자명 목록 (쉼표 구분 또는 대표 저자) |
| `publisher` | string | 출판사 | 출판사명 |
| `publish_date` | string | 출판일 | YYYY-MM-DD 포맷 변환 수집 |
| `discount_rate` | string | 할인율 | 예: `10%` |
| `sale_price` | int | 판매 가격 | 실제 결제 금액 (콤마 제거 정수형) |
| `original_price` | int | 정가 | 할인 전 도서 정가 (콤마 제거 정수형) |
| `point` | string | 적립 포인트 | 예: `540원` 또는 `5%` 적립 포인트 수치 |
| `sale_index` | int | 판매지수 | 교보문고 판매적중도 혹은 클로버 지수 값 활용 |
| `review_count` | int | 리뷰 수 | 작성된 서평/리뷰 개수 |
| `rating` | float | 평점 | 회원 평점 (5.0 혹은 10.0 만점 기준을 10.0 스케일로 통일 권장) |

## 3. 기술 설계 및 수집 방식

### 3.1 기술 스택
- **언어**: Python (버전은 워크스페이스의 공통 `.venv` 환경 기준)
- **핵심 라이브러리**:
  - `requests`: HTTP 요청 송신 및 API 데이터 수집
  - `pandas`: 수집 데이터의 정제 및 CSV 파일 저장
  - `BeautifulSoup` (bs4): API 미작동 시 HTML 파싱 대체용
  - `random`, `time`: 딜레이 제어 및 차단 방지

### 3.2 수집 프로토콜
1. **API 기반 우선 시도**:
   - 웹 브라우저 개발자 도구(F12)의 Network 탭을 통해 교보문고 베스트셀러 페이지 조회 시 발생하는 XHR/Fetch API 호출을 분석합니다.
   - 분석 결과 획득한 API 엔드포인트(예: `https://product.kyobobook.co.kr/api/gw/pub/pnd/best-seller/online?categoryType=33`)로 직접 `requests.get`을 보내 JSON 데이터를 획득합니다.
2. **HTML 파싱 대체**:
   - API 요청 시 CSRF 토큰이나 서명(Signature) 검증 등으로 인해 접근이 막힐 경우, `BeautifulSoup`을 사용하여 도서 목록 HTML 페이지를 직접 파싱합니다.
   - 만약 클라이언트 렌더링에 의한 동적 데이터 로딩 문제가 심각할 경우, 차후 대안으로 `Playwright` 사용을 검토합니다.

### 3.3 안정성 및 서버 보호 대책
- **요청 간격(Sleep)**: 페이지 전환 및 API 요청 사이에 `0.1~0.5초` 사이의 무작위(Random) 대기 시간을 적용하여 서버 트래픽 부하를 방지하고 봇 탐지를 회피합니다.
- **User-Agent 설정**: 실제 브라우저와 유사한 User-Agent 및 관련 헤더 정보를 설정하여 요청합니다.

## 4. 예외 및 오류 처리 계획

- **네트워크 오류**: HTTP 상태 코드(예: 403, 404, 500 등) 감지 시 해당 요청을 건너뛰고 오류 로그를 출력한 뒤 다음 수집을 재개합니다.
- **포맷팅 예외**: 데이터 내 탭(`\t`), 줄바꿈(`\n`, `\r`) 등 CSV 구조를 깨뜨릴 수 있는 유해 문자열은 공백으로 치환하여 저장합니다.
- **데이터 유실**: 수집 실패 혹은 평점 등의 정보 누락 시, 적절한 기본값(예: 평점 0.0, 가격 0)을 할당하여 전체 수집 흐름이 중단되지 않도록 방어 코드를 적용합니다.

## 5. 수집 파이프라인 구현 로드맵

1. **[분석 단계]** 교보문고 카테고리(컴퓨터/IT) 페이지 API 요청 구조 및 파라미터 분석
2. **[설계 검증]** 가벼운 API 호출 테스트 스크립트 작성 및 데이터 정상 수신 여부 확인
3. **[코드 작성]** `kyobobooks/src/scraper.py` 파일 생성 및 구현 (한국어 docstring 및 상세 주석 포함)
4. **[테스트 및 정제]** 소량 데이터 수집 테스트 후 파일 생성 확인
5. **[수집 실행]** 전체 베스트셀러 페이지 순회 수집 후 `bestsellers.csv` 저장
