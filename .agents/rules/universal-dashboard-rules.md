# 💻 범용 대시보드 웹앱 프로토콜 (Universal Dashboard Rules)

본 룰셋은 가공 정제된 분석 데이터를 최종 사용자가 인터랙티브하게 탐색할 수 있도록 대시보드 웹 애플리케이션을 기획, 설계, 빌드할 때 따라야 하는 표준 규칙입니다.

---

## 1. 프론트엔드 아키텍처 및 무결성 수칙 (React Standard)
기술 스택은 유연하게 설정하되, React + TS를 표준 추천 스택으로 활용하는 경우 아래 규칙을 강제 적용합니다.
* **TypeScript 무오류 검증**: 프로덕션 빌드 전 반드시 `tsc -b` 또는 타입 체커를 구동하여 TypeScript 컴파일 에러가 단 1개도 존재하지 않음을 확인해야 합니다.
* **Chart.js의 반응성 처리 (인스턴스 파괴 & 재구동)**:
  - 전역 테마(Theme Selector) 또는 필터에 의해 차트가 변경될 때, 이전 캐싱 리소스로 인해 갱신이 씹히는 현상이 발생합니다.
  - 이를 우회하기 위해 차트 컴포넌트 렌더링 시 **`<Bar key={theme + data.length} ... />`** 와 같이 고유 `key` 속성을 부여하여 런타임에 인스턴스를 즉각 파괴하고 동적 데이터로 재생성하도록 구현합니다.
* **Chart.js 복합 컨트롤러 명시 등록**:
  - 복합 콤보 차트(Bar-Line 등) 사용 시, `LineController` 와 `BarController` 모듈을 반드시 상단 `ChartJS.register(...)` 에 명시적으로 기입하여 빌드 최적화(Tree Shaking) 단계에서 에셋이 유실되어 프로덕션에서 크래시가 발생하는 참사를 예방합니다.

---

## 2. 반응형 인터랙티브 및 요약 지표 설계
* **실시간 런타임 필터 제어**: 검색창, 출판사/카테고리 드롭다운, 가격 범위 슬라이더 등의 필터 상태 변동 시, 리액트 `useMemo` 를 활용하여 런타임 지연 없이 데이터 필터링을 동적 수행해야 합니다.
* **통계 요약 테이블 탑재**: 필터에 의해 필터링된 도서 수량, 가격 평균, 평점 평균 등의 요약 핵심 지표가 화면 상단/하단 표에 실시간으로 다시 계산(Recalculation)되어 반사되도록 연동합니다.

---

## 3. GitHub Pages 배포 및 상대경로 보장
* **상대경로 빌드 고정 (`base`)**: 깃허브 페이지 호스팅(`/repository-name/` 서브폴더 구조) 배포 시 정적 리소스 로딩 404를 완전히 격파하기 위해, Vite 환경 파일(`vite.config.ts`)에 **`base: './'`** (상대경로) 설정을 필히 기입 및 깃허브 master에 커밋 적용합니다.
* **clean-build 배포**: 배포본을 `gh-pages` 브랜치에 푸시할 때, 프로젝트 소스나 `node_modules` 가 오염되어 올라가지 않도록 빌드 산출물(`dist`)만 골라내서 깨끗하게 force push 하도록 배포 워크플로우를 구성합니다.
