# 📊 범용 탐색적 데이터 분석 및 리포팅 프로토콜 (Universal Analysis Rules)

본 룰셋은 수집된 원천 데이터를 분석하고 시각화 이미지와 상세 보고서/슬라이드를 생산할 때 준수해야 하는 분석 표준 가이드라인입니다.

---

## 1. 데이터 분석 및 시각화 필수 수칙 (EDA & Visualization)
* **시각화 라이브러리 사용 제한**: matplotlib 및 pandas 내장 plot을 기반으로 사용하되, **seaborn의 스타일 설정(`sns.set_theme()` 등)은 절대 사용하지 않습니다.**
* **한글 폰트 깨짐 예방**: 모든 파이썬 시각화 스크립트의 도입부에는 반드시 `import koreanize_matplotlib`을 추가하여 한글 폰트가 정상 출력되도록 보장합니다.
* **통계 요약 검증**: 분석 시작 시 원천 데이터의 결측치(null), 자료형(dtype), 유일값(unique count)을 정밀 정합하고 요약 통계(`df.describe()`)를 텍스트 형식으로 문서화합니다.

---

## 2. 보고서 문서 포맷팅 규정 (Reporting Formats)
분석 리포트는 Markdown을 기본으로 작성하되, 필요 시 docx 또는 PDF 다중 배포본을 구성할 수 있는 가이드라인을 충족해야 합니다.
1. **Markdown 보고서 (`.md`)**:
   - 단일 페이지 당 1개의 `<h1>` 구조를 유지합니다.
   - 통계 수치 표는 마크다운 표(`|---|`)로 상세하게 기술합니다.
   - 시각화 이미지는 `images/` 하위에 넘버링하여 배치하고, 본문에 상대경로로 참조 링크(`![Title](images/...)`)를 삽입합니다.
2. **Word 보고서 (`.docx`) / PDF**:
   - `python-docx` 라이브러리를 사용해 Markdown을 문단 스타일(Heading, Title, Normal)로 동적 래핑 및 변환하는 파이썬 변환 유틸리티를 프로젝트 내에 갖춥니다.
   - 변환 시 머리말, 바닥글, 페이지 번호를 삽입해 인쇄 품질을 확보합니다.

---

## 3. 프레젠테이션 슬라이드 규정 (Marp Markdown)
발표 자료 생성 요청 시, Marp Markdown 포맷(`Marp: true`)을 필수 활용하여 작성합니다.
* **디자인 가이드**: 
  - 각 슬라이드는 `---` 구분자로 명확히 구분합니다.
  - 가로 넓이 16:9 (`size: 16:9`) 표준 비율을 정의합니다.
  - 테마 변수(`theme: gaia` 또는 커스텀 css 변수)를 적용하여 슬라이드 전반의 비주얼 일관성을 보장합니다.
